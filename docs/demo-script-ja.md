# デモ脚本（日本語）

## シナリオ

高齢の利用者は、運転免許を返納したあとにスーパーや病院へ行けるか不安を感じています。家族は、返納を急がせるのではなく、まず車なし生活を小さく試せるか確認したいと考えています。

## デモ手順

1. 家族がアプリを開きます。画面タイトルは「車なし生活リハーサル」です。
2. 家族が自宅、よく行く場所、歩ける時間、乗り換え許容を設定します。
3. アプリが車なし生活の成立度を診断します。結果は `ok`、`caution`、`support_needed`、`unknown` で表示します。
4. アプリが最大3つのリハーサルを提案します。
5. 後日、高齢の利用者が「スーパーに行きたい」と言います。
6. アプリが短い外出メモを読み上げ、同じ内容を画面にも表示します。
7. 家族が地図と「家族向けレポート」で支援が必要な場所やデータ不足を確認します。

## 強調点

- 免許返納を決めるアプリではありません。
- ルートはLLMではなく、fixture/mock routerまたは将来のOTP/GTFSから取得します。
- 不足データは「判定不能」または警告として表示します。
- 第一段階はfixture/mock routerで製品閉ループを確認します。
- 生データやAPIキーは配布しません。
