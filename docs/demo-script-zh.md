# 演示脚本（中文）

## 场景

一位高龄用户正在考虑自愿返还驾驶证，但担心返还后无法去超市、医院、药局和市役所。家属不希望应用替老人做决定，而是希望先模拟“没有私家车的日常生活是否还能成立”。

## 演示步骤

1. 家属打开 App，看到「車なし生活リハーサル」。
2. 家属输入自宅地址，选择常去地点，并选择步行承受时间和换乘承受度。
3. App 诊断车なし生活成立度，显示 `ok`、`caution`、`support_needed`、`unknown`。
4. App 生成最多 3 个预演任务，优先选择容易尝试的超市或药局。
5. 返还驾驶证后，老人进入日常模式，说「スーパーに行きたい」。
6. App 朗读简单外出 memo，并在屏幕显示同一内容。
7. 家属查看地图和报告，确认自力可达、条件付き、需要支援、数据不足和下一次预演候选。

## 演示重点

- 这不是驾驶证返还宣传页。
- 这不是通用路线规划器。
- 这不是医疗/法律/护理决策工具。
- 第一阶段用 fixture + mock router 完成产品闭环。
- 第二阶段再接 OTP/GTFS/ODPT，避免一开始陷入数据泥潭。
